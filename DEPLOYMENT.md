# Deployment Guide - SkillSwap Platform

This guide walks you through setting up and deploying the MERN stack SkillSwap platform using **MongoDB Atlas** (Database), **Render** (Backend API & Socket server), and **Vercel** (Frontend UI).

---

## 1. Database Setup: MongoDB Atlas

1. **Sign Up / Log In**: Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) and create a free account.
2. **Create Cluster**: Deploy a new free cluster (Shared tier, AWS/GCP, close region).
3. **Database Access User**:
   - Go to **Database Access** -> **Add New Database User**.
   - Select **Read and write to any database**.
   - Choose username & password (keep note of these).
4. **Network Access (IP Whitelist)**:
   - Go to **Network Access** -> **Add IP Address**.
   - Choose **Allow Access From Anywhere** (`0.0.0.0/0`) so Render instances can connect to your database.
5. **Get Connection String**:
   - Go to **Database** (clusters overview) -> click **Connect**.
   - Choose **Drivers** (Node.js).
   - Copy the connection string. It will look like this:
     ```
     mongodb+srv://<username>:<password>@cluster0.xxxx.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0
     ```
   - Replace `<password>` with your database user password and specify the database name (e.g., `skillswap` before `?`).

## 2. Backend Deployment: Render

Render is excellent for hosting Node.js servers with WebSockets.

1. **Prepare Backend Code**:
   - Ensure the server listens to `process.env.PORT` (defined in `backend/server.js`).
   - Push your code to a Git repository (GitHub/GitLab).
2. **Create Web Service on Render**:
   - Go to [Render Dashboard](https://dashboard.render.com/) and click **New** -> **Web Service**.
   - Connect your GitHub repository.
3. **Configure Settings**:
   - **Name**: `skillswap-backend`
   - **Root Directory**: `backend` (if you push a monorepo containing both folders, set this to `backend`. If you pushed the backend separately, leave blank).
   - **Environment**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `node server.js`
4. **Configure Environment Variables**:
   - Click **Advanced** and add the following keys:
     - `PORT`: `5000` (Render overrides this automatically, but good practice to define)
     - `MONGO_URI`: Your copied MongoDB Atlas connection string.
     - `JWT_SECRET`: A secure random string (e.g. `2d93e1b7a641ef61bd...`)
     - `NODE_ENV`: `production`
5. **Deploy**:
   - Click **Create Web Service**.
   - Note the URL generated by Render (e.g., `https://skillswap-backend.onrender.com`). This will be your `VITE_API_URL` and `VITE_SOCKET_URL` for the frontend.

## 3. Frontend Deployment: Vercel

Vercel is optimized for deploying Vite React applications.

1. **Prepare Frontend Code**:
   - Ensure `frontend/src/context/AuthContext.jsx` and `SocketContext.jsx` read API and Socket endpoints from `import.meta.env.VITE_API_URL` and `import.meta.env.VITE_SOCKET_URL`.
2. **Create Project on Vercel**:
   - Go to [Vercel Dashboard](https://vercel.com/) and click **Add New** -> **Project**.
   - Connect your Git repository.
3. **Configure Settings**:
   - **Root Directory**: Select `frontend` (since it's inside the monorepo).
   - **Framework Preset**: `Vite`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
4. **Add Environment Variables**:
   - Add the following keys in Vercel settings:
     - `VITE_API_URL`: `https://skillswap-backend.onrender.com/api` (replace with your Render backend API URL)
     - `VITE_SOCKET_URL`: `https://skillswap-backend.onrender.com` (replace with your Render backend socket URL)
5. **Deploy**:
   - Click **Deploy**. Vercel will build the React app and give you a public URL (e.g., `https://skillswap.vercel.app`).

## 4. Verification Check

- Visit your Vercel URL.
- Register a new account.
- Search for mentors, request a session, and test the real-time chat functionality.
- Monitor console and server logs to verify database read/writes.
